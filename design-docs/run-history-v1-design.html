Overview

This design delivers the minimal v1 "Run History + Basic Stats" feature: an in-memory (session-only) log of completed simulation runs.

Runs are automatically recorded when you click Reset or when the population naturally reaches zero. Each entry captures:

  • Run number
  • Duration (seconds, derived from ticks)
  • Longest single-agent survival time in that run
  • Final population

The UI extends the existing right sidebar (below the v0.1 hunger sliders) with a prominent "Best longest survival this session" callout + a simple chronological list + Clear button.

Everything stays radically simple — plain lets + one new field on Agent + hardcoded Tailwind in the sidebar. No new files. No persistence.

Key Decisions

1. Record on BOTH Reset and natural population==0 (with guard flag)
Captures the "sole survivor" / lucky agent dynamics that motivated the feature in FUTURE_FEATURES.md. One tiny 3-line check + flag prevents double-recording.
2. "Best longest survival this session" is prominent (emerald, top of section)
Highest-signal value from the experiments. Placed first for immediate value without scanning the list.
3. UI lives inside the existing w-72 right sidebar (below hunger controls)
Lowest friction and risk. Exact reuse of the v0.1 sidebar pattern that just succeeded. Zero layout changes to the flex/canvas relationship.
4. Duration & survival use tick-derived seconds (tick / 60)
Perfect consistency with the existing discrete engine. No extra Date or performance.now() complexity.
5. Only one new field on Agent: birthTick
Matches the "plain lets + minimal model change" philosophy that worked for v0.1 sliders.
6. History list uses innerHTML on a small scrollable div (max-h-28)
Zero dependencies. Fine for a session-only list that rarely exceeds 10–15 entries.

STEP 4 — Add recordCurrentRun() function

Location: Add as a new top-level function near the other control functions (e.g. after resetSim).
NEW FUNCTION
function recordCurrentRun() {
    if (currentRunEnded || runNumber === 0) return;

    // Include still-alive agents at the moment of recording
    let maxLife = currentRunMaxSurvivalTicks;
    for (let a of agents) {
        const life = tick - a.birthTick;
        if (life > maxLife) maxLife = life;
    }

    const durationTicks = Math.max(0, tick - runStartTick);
    const durationSec = Math.round(durationTicks / 60);

    const longestSec = Math.round(maxLife / 60);
    const finalPop = agents.length;

    runHistory.push({
        run: runNumber,
        durationSec: durationSec,
        longestSec: longestSec,
        finalPop: finalPop
    });

    if (longestSec > bestLongestSec) bestLongestSec = longestSec;

    currentRunEnded = true;
    console.log(`[Run History] Recorded Run #${runNumber} — ${durationSec}s, longest ${longestSec}s, pop ${finalPop}`);
}

STEP 5 — Update resetSim() + add startNewRun()

REPLACE resetSim + ADD HELPER
function startNewRun() {
    runNumber++;
    runStartTick = tick;
    currentRunMaxSurvivalTicks = 0;
    currentRunEnded = false;
}

function resetSim() {
    recordCurrentRun();     // record what just ended
    tick = 0;
    init();
    startNewRun();          // begin fresh run tracking
}

STEP 7 — Add renderRunHistory() + clearRuns()

NEW RENDER + CLEAR FUNCTIONS
function renderRunHistory() {
    const bestEl = document.getElementById('best-longest');
    const listEl = document.getElementById('run-list');
    if (!bestEl || !listEl) return;

    bestEl.textContent = bestLongestSec > 0 ? bestLongestSec + 's' : '—';

    if (runHistory.length === 0) {
        listEl.innerHTML = '
No runs yet
';
        return;
    }

    let html = '';
    for (let i = runHistory.length - 1; i >= 0; i--) {   // newest first
        const r = runHistory[i];
        html += `

            Run #${r.run}
            ${r.longestSec}s
            ${r.durationSec}s • ${r.finalPop}
        
`;
    }
    listEl.innerHTML = html;
}

function clearRuns() {
    runHistory = [];
    bestLongestSec = 0;
    renderRunHistory();
}
