Design: Live Parameter Configuration Window + Hunger & Consumption Tuning for Punishing Starvation (Phase 0 Bare Survival)

Project: trait-evolution-sim
Author: Grok (AI Design Assistant, following AGENTS.md)
Date: 2026-05-27
Status: Draft
Current Phase: v0.1 Foundation — Bare Survival (Phase 0)

Overview

This design introduces two tightly coupled changes for the Phase 0 foundation:

  1. A live, extensible parameter configuration panel integrated into the existing right sidebar. This replaces scattered magic numbers with a single CONFIG source of truth and provides immediate visual + interactive controls.
  2. Tuning of the hunger and consumption system to make starvation meaningfully punishing.

Background & Motivation

Current State

  • Simulation loop: gameLoop() → update() → draw()
  • Agent hunger increases by 0.035 per tick, dies at 100
  • Self-consume relief: -1.6, Ground food relief: -2.5
  • Baseline survival without eating ≈ 48 seconds

Pain Points

  • Starvation is too forgiving
  • No easy way to tune values during development
  • Magic numbers scattered everywhere — bad for future phases

Goals & Non-Goals

Goals

  • Centralize all scalars behind a CONFIG object
  • Visible, always-on parameter panel in the sidebar
  • Most values update live
  • Make starvation punishing while remaining playable
  • Easy to extend for future phases

Proposed Design Highlights

New Punishing Defaults

CONFIG.hunger = {
  rate: 0.052,                 // +49% faster
  selfConsumeRelief: 1.25,     // 22% less relief
  groundEatRelief: 1.95,       // 22% less relief
};

Expected survival time drops significantly, forcing better foraging behavior even with random movement.

Alternatives Considered

  • Floating modal (rejected — breaks liveness)
  • Bottom tuner console (rejected — too big layout change)
  • JSON textarea (too advanced for now)

Chosen: In-sidebar panel with schema-driven controls.

Key Decisions

  • Mutable CONFIG + schema for long-term maintainability
  • Live-by-default with clear "Reset Required" badges
  • Focus on rate + relief scalars (no new mechanics)
  • No persistence in this increment

PR Plan (8 Incremental Steps)

Each step is small, safe, and reviewable. Follow one per focused session.

  1. Pure refactor: Introduce CONFIG + replace literals (no UI)
  2. Add CONFIG_SCHEMA and helper scaffolding
  3. Add static sidebar panel HTML shell
  4. Wire first live hunger controls
  5. Complete wiring for all parameters
  6. Apply punishing defaults (the real behavior change)
  7. Polish, defaults button, comments
  8. Optional: test extensibility with 1-2 new params
